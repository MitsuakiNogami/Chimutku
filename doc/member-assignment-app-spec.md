# メンバー割り振りアプリ 仕様まとめ

## 目的

複数人の参加者を、ボーリング、ダーツ、その他イベントの決められたフォーマットに合わせてランダムに割り振るアプリケーション。

競技ごとの専用ロジックに閉じず、基本的には以下の考え方で設計する。

- 参加者を入力する
- フォーマットを選ぶ
- フォーマット内の枠に参加者を割り振る
- 結果を確認、コピー、保存、再利用する

## 基本モデル

```ts
type Participant = {
  id: string;
  name: string;
  active: boolean;
};

type Slot = {
  id: string;
  label: string;
  capacity: number;
};

type Format = {
  id: string;
  category: "bowling" | "darts" | "custom";
  name: string;
  slots: Slot[];
};

type Assignment = {
  slotId: string;
  slotLabel: string;
  capacity: number;
  participants: Participant[];
};
```

## テンプレートカテゴリ

テンプレートはタブでカテゴリ分けする。

- ボーリング
- ダーツ
- カスタム

各タブを選択すると、そのカテゴリに属するテンプレートのみを表示する。

## ボーリング仕様

ボーリングでは、レーン数と各レーンごとの人数を変更できる。

例:

```txt
レーン数: 3

1レーン人数: 2
2レーン人数: 4
3レーン人数: 3
```

この場合、合計9名までを各レーンに割り振る。

参加者が定員を超える場合は、余った参加者を `待機` として表示する。

## ダーツ通常テンプレート

ダーツでは以下の通常テンプレートを持つ。

- ダーツリーグ戦 13試合
- 関東大会 5試合
- ダーツ 01 ダブルス
- クリケット トリオス

## ダーツリーグ戦フォーマット

ダーツリーグ戦では、第1戦から第13戦までの試合形式が固定されている。

人数ルール:

- シングル: 1名
- ダブルス: 2名
- トリオス: 3名
- ガロン: 4名

試合構成:

```txt
第1戦  701 トリオス
第2戦  クリケット トリオス
第3戦  501 ダブルス
第4戦  501F ダブルス
第5戦  701 シングル
第6戦  クリケット シングル
第7戦  901 トリオス
第8戦  クリケット トリオス
第9戦  クリケット ダブルス
第10戦 501F シングル
第11戦 501 シングル
第12戦 クリケット シングル
第13戦 1101 ガロン
```

合計は延べ27出場枠。

## 関東大会フォーマット

関東大会では、第1戦から第5戦までの試合形式が固定されている。

試合構成:

```txt
第1戦 901 トリオス
第2戦 クリケット トリオス
第3戦 501F ダブルス
第4戦 クリケット ダブルス
第5戦 1101 ガロン
```

合計は延べ14出場枠。

関東大会テンプレートも、ダーツリーグ戦と同じく複数出場型のフォーマットとして扱う。

## ダーツリーグ戦の制約

ダーツリーグ戦では以下の制約を加味する。

- 1人が参加できる最大ゲーム数は7ゲームまで
- シングルは1人最大2ゲームまで
- 同じ試合内で同じ人を重複させない
- 出場回数が少ない人を優先して割り振る
- 制約上どうしても埋められない場合は `空き` として表示する

## ダーツリーグ戦の前半・後半

ダーツリーグ戦では、試合を前半・後半に分ける。

- 第1戦から第6戦まで: 前半
- 第7戦から第13戦まで: 後半

メンバーごとに、前半・後半それぞれで優先設定を持てる。

優先設定:

- 指定なし
- 01優先
- クリケ優先

割り振り時には、ゲーム種別に応じて優先設定を重みとして加味する。

ゲーム種別の判定:

- クリケットを含む試合: クリケット
- それ以外: 01

## ダーツリーグ戦の個別編集

ダーツリーグ戦では、割り振り後に各枠を個別に変更できる。

各出場枠はプルダウンで表示され、以下が可能。

- 別メンバーに入れ替える
- 空きにする
- 空き枠にメンバーを入れる

手動変更時にも以下をチェックする。

- 同じ試合内で同じ人を重複させない
- 1人最大7ゲームを超えない
- シングル最大2ゲームを超えない

## カスタム仕様

カスタムでは以下を指定できる。

- 枠数
- 1枠あたりの人数

例:

```txt
枠数: 5
1枠あたりの人数: 4
```

この場合、5枠に4名ずつ割り振る。

## 割り振りロジック

通常フォーマットでは、以下の流れで割り振る。

1. 参加者を取得
2. Fisher-Yates方式でシャッフル
3. 各枠の定員に従って順番に割り振る
4. 入りきらない参加者は待機にする

ダーツリーグ戦では、参加者を複数試合に割り当てる。

1. 各参加者の総出場数を管理
2. 各参加者のシングル出場数を管理
3. 試合ごとに候補者を抽出
4. 上限を超えない候補者から選ぶ
5. 前半・後半の優先設定を重みとして加味する
6. 出場数が少ない人を優先する

## 履歴機能

抽選結果を履歴として保存できる。

保存する内容:

- 保存日時
- カテゴリ
- フォーマットID
- フォーマット名
- 参加者入力内容
- カスタム設定
- ボーリング設定
- ダーツリーグ優先設定
- 割り振り結果

履歴では以下ができる。

- 現在の結果を履歴保存
- 過去の履歴を閲覧
- 過去の結果を表示して再利用
- 履歴を個別削除
- 履歴を全削除

履歴は `localStorage` に保存する。


- ボーリングのレーン数変更

- ダーツリーグ戦13試合フォーマット

- ダーツリーグ戦の前半・後半優先設定

- 再抽選
- 結果コピー
- 履歴保存
- 履歴閲覧
- 履歴再利用
